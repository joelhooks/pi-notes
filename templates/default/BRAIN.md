# pi-notes brain

Use this repo's `.brain/` as durable project memory, not a transcript dump.

- Organize by usefulness: Projects, Areas, Resources, Archives.
- Keep notes atomic, linked, and source-grounded.
- Capture durable decisions, terms, tradeoffs, gotchas, sources, questions, receipts, and review feedback.
- Use `.brain/data/**` for large structured source data.
- Use `.brain/components/**` for reusable Svelte renderers in MDSvX notes.
- After Brain edits, run `pi-notes brain check` or this repo's Brain check script.

Ask: where will this be useful next?
